## What this is

A scientific software pipeline for detecting localized anomalies — bubble-collision discs and cosmic-string discontinuities — in CMB temperature maps. Built for reproducibility, honest sensitivity reporting, and methodological publication.

What it does **not** do: claim detections. Every candidate is tested against positive controls, null controls, injection-recovery curves, and global p-values before being reported.

## Quick start

```bash
# Install dependencies
pip install healpy numpy pyyaml typer rich pydantic pytest

# Initialize a config and run a quick demo
python pipeline.py init --quick --mechanism bubble

```

One command produces: filtered map, candidate list, recovery curves, figures, and `validation_report.md`.

## Pipeline structure

```
pipeline.py init      →  pipeline.yaml          (validated config)
pipeline.py generate  →  runs/{id}/T_map.npy    (CMB map + optional injection)
pipeline.py scan      →  runs/{id}/candidates.csv
pipeline.py validate  →  runs/{id}/validation/  (positive/null/recovery)
pipeline.py report    →  runs/{id}/validation_report.md + figures/
pipeline.py run       →  all steps in sequence
pipeline.py demo      →  one-command end-to-end demo
```

Each step is idempotent, outputs structured JSON metadata, and is independently testable.

## Detection filter

We employ a three-step bandpass matched-filter strategy designed for localized CMB-like signatures in the presence of CMB confusion noise.

1. **Spectral prewhitening** — the input map is divided by √C_eff(ℓ) in harmonic space, suppressing the dominant large-scale CMB power and flattening the noise spectrum.

2. **Scale-matched bandpass** — a top-hat filter is applied around the multipole range most relevant to the template radius (ℓ_peak = π/r_rad, band width factor 3×), reducing confusion from unrelated angular scales.

3. **Local disc-versus-annulus statistic** — candidate amplitudes are evaluated as disc_mean / annulus_std, which provides robustness against mask boundaries, local background fluctuations, and residual foreground-like structure.

We tested pure Wiener filtering as an alternative, but found that in the single-frequency, CMB-confusion-dominated regime it does not provide the expected sensitivity gain. The implemented prewhitened bandpass filter therefore represents a pragmatic matched-filter approximation for the validation setting considered here. Under controlled simulations, this strategy reduces the effective sensitivity threshold relative to the simple disc-response detector while preserving transparent null-control and injection-recovery behavior.

**Sensitivity note:** At nside=128, 7.3′ beam, 35 µK·arcmin noise, the empirical CMB confusion threshold is ~300 µK for an 8° bubble disc. In the instrument-noise-only limit (ILC-cleaned multi-frequency maps), the same pipeline achieves ~3–10 µK — this is the target configuration for Planck and future LiteBIRD data.

## Signal models

### Bubble collision

Motivated by eternal inflation scenarios in which our Hubble volume collided with a neighbouring bubble. Predicted signature: a circular disc-like anomaly in CMB temperature with enhanced rim, sub-degree edge width, and E-mode polarization excess with near-zero B-mode.

Template parameters: amplitude (µK), angular radius (deg), edge width (deg), rim boost factor.

### Cosmic string

A one-dimensional topological defect producing a linear temperature discontinuity (Kaiser-Stebbins effect). Predicted signature: a step-function discontinuity across the sky with characteristic E/B polarization ratio.

Template parameters: amplitude (µK), length (deg), orientation (deg), width (deg).

## Validation

Every pipeline run produces three mandatory validation outputs.

### Positive control

Injects a signal with amplitude = 5× background map RMS and verifies recovery rate ≥ 80%. A failure here indicates a bug in the template, beam convolution, or map generation — not a sensitivity limitation.

### Null control

Scans pure background maps (no signal) at random positions. False-positive rate must remain below 5%. Sets the empirical detection threshold for the recovery grid.

### Injection-recovery grid

Sweeps over all (amplitude, radius) combinations with n_trials random sky positions each. Outputs:

- `recovery_matrix.csv` — trial-level results
- `recovery_summary.csv` — recovery rate + 95% Wilson CI per cell (paper-ready)
- `fig_recovery_heatmap.png` — visual summary
- `fig_recovery_curves.png` — recovery rate vs amplitude per radius
- `fig_sensitivity.png` — minimum detectable amplitude at 50% recovery

## Reproducing results

All outputs are deterministic given the same `seed` and config. To reproduce exactly:

```bash
# Freeze input snapshots
python scripts/freeze_inputs.py

# Strict reproduction from frozen inputs
python scripts/repro_from_snapshot.py --config artifacts/run_config.yaml
```

Each run archives `meta_*.json` files with full parameter provenance. The `validation_report.md` includes the exact command to reproduce.

## Candidate vetting (v0.4)

Two systematics tests separate cosmological candidates from foregrounds and flukes:

**Frequency consistency** — a true CMB-frame signal is achromatic; galactic dust rises ~145× from 143 to 353 GHz (modified blackbody, β=1.55, T=19.6 K). The test measures candidate SNR at 100/143/217/353 GHz and fits the log-slope: flat → pass, rising → dust (reject), falling → synchrotron (reject).

**Half-mission split consistency** — a sky signal appears identically in both Planck half-mission maps; a noise fluke or scan systematic appears in one. The test requires SNR ≥ threshold/√2 in both halves with consistent values.

```bash
# Vet a candidate (simulation mode):
python pipeline.py vet pipeline.yaml --lon 45 --lat 45 --radius 8 --simulate-signal 500
# → Frequency: PASS (achromatic, slope≈0) 

python pipeline.py vet pipeline.yaml --lon 45 --lat 45 --radius 8 --simulate-dust 800
# → Frequency: REJECT (dust-like, slope≈2.2)
```

**Real Planck data** — `scripts/download_planck.py --set foreground` fetches the public PLA products; `src/data/planck_loader.py` loads, converts to µK, and degrades to pipeline resolution.
